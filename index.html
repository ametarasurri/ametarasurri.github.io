Overview & Methodology

Understanding the data pipeline from raw Kaggle dataset to actionable insights

🗃️

Dataset

  • Source: Kaggle — Indian Startup Funding
  • Size: 3,044 rows × 9 columns
  • Coverage: 2015 – mid 2020
  • Key fields: startup_name, date, industry_vertical, city, investors_name, investment_type, amount_usd
🧹

Data Cleaning

  • Inconsistent city names: Bengaluru and Bangalore treated as separate cities; Gurugram vs Gurgaon; New Delhi vs Delhi. All standardised to single canonical values (Bangalore, Gurgaon, Delhi).
  • Non-numeric funding amounts: amount_usd was stored as text with commas (e.g. 20,00,00,000). Commas were removed and the column was cleaned and converted to a numeric (REAL) data type for aggregation.
  • Inconsistent industry labels: eCommerce, ECommerce, and E-commerce all appeared as separate values. Standardised uniformly to "E-Commerce" for accurate sector-level analysis.
  • Redundant column dropped: The remarks column was 86% null and contained no analytically useful information — removed to reduce noise.
  • Preprocessing pipeline: Data was cleaned using Python and Claude AI before being loaded into SQLite for query execution.

Which sectors are growing year over year?

Tracking sector consistency and emergence across the Indian startup ecosystem

CTE DENSE_RANK PARTITION BY substr() 
WITH yearly_sector AS (
    SELECT 
        substr(date, 7, 4) AS year,
        industry_vertica AS sector,
        COUNT(*) AS total_deals,
        ROUND(SUM(amount_usd) / 1000000000, 2) AS funding_billions
    FROM startup_funding_clean
    WHERE industry_vertica IS NOT NULL 
    AND industry_vertica != ''
    AND substr(date, 7, 4) IN ('2016', '2017', '2018', '2019')
    GROUP BY year, industry_vertica
),
ranked AS (
    SELECT 
        year, sector, total_deals,
        funding_billions || 'B' AS funding_billions,
        DENSE_RANK() OVER (
            PARTITION BY year ORDER BY total_deals DESC
        ) AS sector_rank
    FROM yearly_sector
)
SELECT * FROM ranked
WHERE sector_rank <= 5
ORDER BY year, sector_rank;

Top 5 funded startups per major city

Identifying the biggest players in each Indian startup hub

CTE DENSE_RANK PARTITION BY city Multi-condition Filter 
WITH startup_totals AS (
    SELECT 
        startup_name, city,
        COUNT(*) AS total_rounds,
        ROUND(SUM(amount_usd) / 1000000000, 2) AS total_funding_billions
    FROM startup_funding_clean
    WHERE startup_name IS NOT NULL AND startup_name != ''
    AND city IS NOT NULL AND city != ''
    AND amount_usd IS NOT NULL
    GROUP BY startup_name, city
),
ranked AS (
    SELECT 
        startup_name, city, total_rounds,
        total_funding_billions || 'B' AS total_funding,
        DENSE_RANK() OVER (
            PARTITION BY city ORDER BY total_funding_billions DESC
        ) AS rank_in_city
    FROM startup_totals
)
SELECT * FROM ranked
WHERE rank_in_city <= 5
AND city IN ('Bangalore', 'Mumbai', 'Delhi', 'Gurgaon', 
             'Pune', 'Hyderabad', 'Chennai', 'Noida')
ORDER BY city, rank_in_city;

Month over Month funding trends

Tracking monthly funding fluctuations to detect patterns and anomalies

CTE LAG MoM % Change substr() 
WITH monthly_totals AS (
    SELECT 
        substr(date, 7, 4) AS year,
        substr(date, 4, 2) AS month,
        COUNT(*) AS total_deals,
        ROUND(SUM(amount_usd) / 1000000000, 2) AS funding_billions
    FROM startup_funding_clean
    WHERE amount_usd IS NOT NULL
    AND substr(date, 7, 4) IN ('2016', '2017', '2018', '2019')
    GROUP BY year, month
),
with_lag AS (
    SELECT 
        year, month, total_deals,
        funding_billions || 'B' AS funding_billions,
        LAG(funding_billions, 1) OVER (ORDER BY year, month) || 'B' AS prev_month_funding,
        ROUND(
            (funding_billions - LAG(funding_billions, 1) OVER (ORDER BY year, month)) 
            * 100.0 / LAG(funding_billions, 1) OVER (ORDER BY year, month), 
        2) AS pct_change
    FROM monthly_totals
)
SELECT * FROM with_lag ORDER BY year, month;

Key Insight

Indian startup funding is highly lumpy — not smooth or seasonal. A single mega-round from a Flipkart or Rapido can distort an entire month's figures by over 1000%. August 2017 spiked +1,517% and August 2019 spiked +1,410%, both driven by individual massive rounds. This makes monthly deal count a far more reliable trend indicator than raw funding amount for tracking ecosystem health. The 2018 slowdown is clearly visible — October 2018 saw a -91% collapse in funding, with the contraction extending through Nov–Dec 2018, reflecting a broader market correction across the Indian startup ecosystem that preceded the selective 2019 recovery.

Which investment stage dominates each city?

Mapping India's startup maturity divide across major cities

CASE WHEN LIKE CTE DENSE_RANK 
WITH city_stage AS (
    SELECT 
        city,
        CASE 
            WHEN investment_type LIKE '%Seed%' THEN 'Early Stage'
            WHEN investment_type LIKE '%Angel%' THEN 'Early Stage'
            WHEN investment_type LIKE '%Pre-Series%' THEN 'Early Stage'
            WHEN investment_type LIKE '%Series A%' THEN 'Growth Stage'
            WHEN investment_type LIKE '%Series B%' THEN 'Growth Stage'
            WHEN investment_type LIKE '%Series C%' THEN 'Late Stage'
            WHEN investment_type LIKE '%Series D%' THEN 'Late Stage'
            WHEN investment_type LIKE '%Series E%' THEN 'Late Stage'
            WHEN investment_type LIKE '%Private Equity%' THEN 'Late Stage'
            ELSE 'Other'
        END AS stage,
        COUNT(*) AS total_deals,
        ROUND(SUM(amount_usd) / 1000000000, 2) AS funding_billions
    FROM startup_funding_clean
    WHERE city IS NOT NULL AND city != ''
    AND investment_type IS NOT NULL AND investment_type != ''
    GROUP BY city, stage
),
ranked AS (
    SELECT city, stage, total_deals,
        funding_billions || 'B' AS funding_billions,
        DENSE_RANK() OVER (
            PARTITION BY city ORDER BY total_deals DESC
        ) AS stage_rank
    FROM city_stage
)
SELECT * FROM ranked
WHERE stage_rank = 1
AND city IN ('Bangalore', 'Mumbai', 'Delhi', 'Gurgaon', 
             'Pune', 'Hyderabad', 'Chennai', 'Noida')
ORDER BY total_deals DESC;

Key Insight

India's startup funding landscape reveals a clear maturity divide. Bangalore leads Late Stage dominance with 417 deals worth $14.19B — confirming its position as home to India's most mature unicorns. Mumbai (280 deals, $4.41B), Gurgaon (164 deals, $3.54B), and Chennai (53 deals, $0.67B) follow the same Late Stage pattern, suggesting established ecosystems backing scaling companies. In contrast, Delhi (273 deals, $0.29B), Hyderabad (60 deals, $0.06B), Pune (58 deals, $0.11B), and Noida (50 deals, $0.06B) are predominantly Early Stage ecosystems — high deal count but low total funding, indicating these cities are nurturing the next generation of startups rather than scaling existing ones. This has direct implications for investors: Bangalore and Mumbai for growth-stage returns, emerging cities for early-stage bets at lower valuations.

Top 4 Cities Hold 72% of All Deals

Bangalore (27.63%), Mumbai (18.63%), Delhi (14.98%), and Gurgaon (11.07%) account for 72.31% of all startup deals. The remaining ~28% is spread thinly across Pune, Hyderabad, Chennai, Noida, and 50+ other cities.

Funding is Lumpy, Not Seasonal
